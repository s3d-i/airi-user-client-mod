# Docs

The docs are split by concern:

- [design/](./design/README.md): target architecture, boundaries, event model, and guardrails
- [dev/](./dev/README.md): current implementation state and contributor-facing development context
- [reference/](./reference/README.md): snapshot-specific implementation references gathered from code reading and external repo research

Read `design/` when you want to understand the intended system shape.

Read `dev/` when you want to understand what exists today and how it differs from the design docs.

Read `reference/` when you want implementation-oriented notes derived from a specific code snapshot, especially when the source of truth lives partly in another repository.
